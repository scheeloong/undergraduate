Server Database

Purpose

The purpose of this project was to implement a secure server database as a solution to Adobe's 2013 Database Hack.

Design Challenges

Hash Table Data Structure

We chosed to use a Hash Table Data Structure as users will only create a new account and edit their accounts on Adobe. They will never delete their account on Adobe. We will never have to scan through all users on the database as well. Therefore, we chosed to use a Hash Table Data Structure.

Encryption

We chosed to use the Solitaire Encryption Algorithm as we wanted to work with a simple encryption algorithm for our presentation.

Result

The secure server database works perfectly as shown in Video 1. Notice that the data is encrypted inside the database but is decrypted when sent back out of the database.
